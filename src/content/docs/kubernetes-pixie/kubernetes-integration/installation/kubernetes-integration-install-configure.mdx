The easiest way to install the Kubernetes integration is to use our guided install to generate a manifest. It bundles not just the integration DaemonSets, but also other New Relic Kubernetes configurations, like [Kubernetes events](/docs/integrations/kubernetes-integration/kubernetes-events/install-kubernetes-events-integration), [Prometheus OpenMetrics](/docs/integrations/prometheus-integrations/get-started/new-relic-prometheus-openmetrics-integration-kubernetes), and [New Relic log monitoring](/docs/logs).

You have these options to install the Kubernetes integration:

* Downloading the [New Relic Kubernetes quickstart](https://newrelic.com/instant-observability/kubernetes). The quickstart includes pre-built resources like dashboards, instrumentations, and alerts.

* Following the [automated and guided install](https://one.newrelic.com/launcher/k8s-cluster-explorer-nerdlet.cluster-explorer-launcher?pane=eyJuZXJkbGV0SWQiOiJucjEtaW5zdGFsbC1uZXdyZWxpYy5ucjEtaW5zdGFsbC1uZXdyZWxpYyIsInBhdGgiOiJndWlkZWQiLCJlbnYiOiJrdWJlcm5ldGVzIiwiaW5pdGlhbEFjdGlvbkluZGV4IjpudWxsLCJhY3Rpb25JbmRleCI6MX0=). If you follow [Kubernetes guided install](https://docs.newrelic.com/docs/kubernetes-pixie/kubernetes-integration/installation/guided-install) you have these options to install the Kubernetes integration:
  * New Relic CLI
  * [Helm](https://onenr.io/0Y8wpoYJJQO). See [Install the Kubernetes integration using Helm](/docs/kubernetes-pixie/kubernetes-integration/installation/install-kubernetes-integration-using-helm).
  * Manifest

As a Kubernetes user, you might consider [Auto-telemetry with Pixie](/docs/auto-telemetry-pixie/get-started-auto-telemetry-pixie) for deeper insights into your Kubernetes clusters and workloads. No language agents required.
